# Silent Help Assessment System version 2.2

## Core idea

We assess **two things at once**:

### 1. Stress type

What kind of stress feels strongest right now?

Possible types:

* **Overwhelmed**
* **Anxious**
* **Frustrated**
* **Sad**
* **Pressure**

### 2. Intensity level

How strong is it right now?

Possible levels:

* **Light**
* **Elevated**
* **Intense**
* **Urgent**

So the final result becomes:

* **Light Overwhelm**
* **Elevated Anxiety**
* **Intense Frustration**
* **Urgent Emotional Support**
* **Intense Performance Pressure**

# FINAL SYSTEM FLOW

## Q1 → identify main stress type

## Q2 → measure intensity + confirm emotional pattern

## Q3 → measure impact / reaction / body state + risk flags

## Result → support path + level + optional safety handling

# STEP 1 — MAIN QUESTION

## Q1

**“What feels strongest for you right now?”**
**Pick the closest one.**

### A.

**“I have too many things to handle.”**
→ Primary type: **Overwhelmed**

### B.

**“I’m worried something might go wrong.”**
→ Primary type: **Anxious**

### C.

**“Something is really frustrating me.”**
→ Primary type: **Frustrated**

### D.

**“I feel low or emotionally drained.”**
→ Primary type: **Sad**

### E.

**“I feel pressure to succeed or perform.”**
→ Primary type: **Pressure**

## What Q1 does

Q1 sets the **starting branch**, but not the full truth.

That means:

* it chooses the main direction
* but Q2 and Q3 can still add hidden secondary signals

So Q1 is the **primary branch selector**, not the only decision-maker.

---

# STEP 2 — BRANCH QUESTIONS

Each branch has:

* one Q2 question
* one Q3 question
* each answer gives:

  * **intensity score**
  * **emotion confirmation signal**
  * optional **secondary emotion signal**
  * optional **risk flag**

# FINAL SCORING SYSTEM

## 1. Intensity score

Use:
**Intensity Average = (Q2 + Q3) / 2**

### Level mapping

* **1.0 – 1.5** → Light
* **1.6 – 2.5** → Elevated
* **2.6 – 3.4** → Intense
* **3.5 – 4.0** → Urgent

This keeps your clean scoring model from the 2.1 version. 

---

## 2. Urgent override rules

This is the main upgrade.

Even if the average is lower, some answers should force a stronger result.

## Override to at least Intense if any:

* `high_physical_activation`
* `conflict_escalation`
* `social_withdrawal`
* `shutdown_state`
* `high_stakes_strain`

## Override to Urgent if any:

* `panic_like_state`
* `loss_of_control`
* `hopelessness_marker`
* `severe_withdrawal`
* `burnout_marker`

So now dangerous or heavier answers do not disappear inside the average.

---

## 3. Emotional result

At the end:

* highest emotion score = **primary type**
* second highest emotion score = **secondary type**

### Example

* primary = anxious
* secondary = overwhelmed

This helps your app feel much smarter.

## Internal result object

Store more detail:

```ts
type SupportType =
  | "overwhelmed"
  | "anxious"
  | "frustrated"
  | "sad"
  | "pressure";

type IntensityLevel =
  | "light"
  | "elevated"
  | "intense"
  | "urgent";

interface AssessmentResultV22 {
  primaryType: SupportType;
  secondaryType?: SupportType;
  level: IntensityLevel;
  supportPath:
    | "overwhelm_support"
    | "anxiety_support"
    | "frustration_support"
    | "emotional_support"
    | "performance_support";
  intensityAverage: number;
  riskFlags: string[];
  shouldShowSafetyBar: boolean;
}
```
